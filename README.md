## セキュリティ関する設計

- パスワードは平文で保存せず、bcryptjs でハッシュ化
- 秘密の質問の回答も同様にハッシュ化
- JWTは jose で発行し、期限付きのトークンとして使用
- ログインAPIは認証に失敗した場合に詳細を非表示
- セッション状態はクライアント側で保存しつつ、認証トークンの有効性を確認
- 秘密の質問は1問ではなく18問全て解答させることで誰も突破できな完璧な圧倒的セキュリティを実現しました

## 認証の流れ

1. ユーザー登録画面で名前・メールアドレス・パスワード・秘密の質問を入力
2. サーバー側でパスワードと回答をハッシュ化して保存
3. ログイン画面でパスワードまたは秘密の質問を使って認証
4. 認証に成功するとJWTトークンは httpOnly クッキーでセットされ、アプリ内でログイン状態を保持
5. 画面上部のヘッダーでログイン済みかどうかが表示

## 実装した場所

- src/lib/userStore.ts
  - src/lib/userStore.ts
  - ユーザー情報の読み込み・保存を管理
  - パスワードおよび秘密の質問の回答を bcryptjs でハッシュ化して保存
  - メールアドレスおよびパスワードの重複登録を防止
  - 開発環境ではユーザー情報を data/users.json に保存
  - ログイン時のパスワード照合と秘密の質問の回答照合を実装

- src/lib/auth.ts
  - JWTを jose で生成・検証
  - 12時間のトークン有効期限を設定
  - 本番環境では JWT_SECRET を必須とし、安全に署名鍵を管理

- src/app/api/login/route.ts
  - メールアドレスとパスワード、または秘密の質問によるログイン認証を実装
  - 認証成功時に JWT を生成し、httpOnly Cookie に保存
  - ログインユーザーの情報を返却

- src/app/api/register/route.ts
  - 登録処理を行い、必須項目が揃っているかチェック

- src/app/api/session/route.ts
  - 既存のセッションを検証し、ログイン状態を返す

- src/app/api/logout/route.ts
  - ログアウト時に token クッキーをクリアする

- src/app/_contexts/SessionContext.tsx
  - Context を利用してログイン状態とユーザー情報を管理
  - 初回表示時にセッションを確認
  - ログアウト時に状態をクリア
